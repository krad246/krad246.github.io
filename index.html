MS-TOS

A real-time preemptive task scheduler designed for the MSP430 and MSP430X CPU architectures written in embedded C. This operating system is highly configurable at build time, allowing users to tweak everything from the timer used for preemption to the size of the task stacks. The system makes use of memory pool allocation for bounded and deterministic response times and the Virtual Time Round-Robin deterministic fair share scheduler for consistent and reliable time slicing over threads. However, it is highly dynamic as much focus has been devoted to amortizing away instantaneous jitter through careful choice of efficient, dynamically growable data structures such as deques and red-black trees.

The operating system features a fixed-priority scheme to handle real-time priority threads interleaved with non real-time threads, allowing for lightning fast real-time response and great general performance for other applications. The system supports tickless operation, mutexes, semaphores, (and other synchronization primitives) thread blocking and sleeping, cooperative scheduling, priority inheritance, and a mixed fixed-priority / Virtual Time Round Robin scheduling algorithm.

EKG Analyzer

This is a real-time QRS peak detection implementation written on the ultra low-power MSP430G2553 board as part of a larger visualization project for the ARCH5424 class at UVa. The visualization project aimed to synchronize real-time EKG curve features with the phases of a simple heartbeat sprite sheet to illustrate the link between the electrical activity of the nervous system and the heart's motion.

This was a from-scratch implementation of the QRS detection algorithm as outlined in DSP literature, translated into embedded C on the very limited MSP430G2553 microcontroller. This system processed EKG samples in real time at about 200 Hz, applying a multitude of high order filters and nonlinear operations, to the input data as it arrived in real-time. Many of these operations would run terribly on the core as it does not feature a hardware multiplier and runs at a paltry 1 MHz, but through careful optimization, I was able to highly optimize this, to the point that every sample was processed in under 1 ms.

PID Heat Controller

This was a temperature controller designed as part of my UVa capstone. Aimed at reducing static power dissipation (and subsequent wear) in heat lamp temperature regulation systems by aggressively pulse-width modulating a heat lamp in a PID control loop conditional on the temperature of the target area. This device synchronized itself to the AC line frequency so as to not add line noise and drive up electricity costs from inrush current, the same inrushes that would also damage a heat lamp.

Making use of an SPI temperature sensor, an I2C LCD screen, the CC3220F LaunchPad, and substantial power electronics surrounding it, this system collected temperature readings, processed them in 32-bit fixed-point arithmetic, and modulated lamp intensity as an integer number of AC half-cycles. At the same time, it also sampled user input from a joystick / button combo and published to a simple website. I developed the whole fixed-point library and the drivers for our external hardware in embedded C, integrating it all in a PID control loop application hosted on TI-RTOS. I also helped with the electronics schematic design and oscilloscope debugging as we pieced the system together.

Red-Black Tree

This is a canonical red-black tree implementation modeled after the Linux rbtree source and covers all basic operations of red-black trees. Designed as an intrusive node meant to be embedded in larger structures, this design decouples memory management from the fundamental behavior of the tree and keeps things simple. This was written in C and tested using GDB.
